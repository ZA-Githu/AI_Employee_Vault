# Skill: Weekly Audit
**ID:** skill-weekly-audit
**Version:** 2.0.0
**Tier:** Gold
**Trigger:**
- Scheduled: every Monday at session start
- Manual: `run weekly audit` | `generate CEO briefing` | `audit this week`
- Partial: `audit tasks` | `audit social` | `audit accounting`

## Description

Runs the full cross-domain weekly audit — tasks, social media, accounting, approvals, and errors — entirely from vault data. Produces an accounting report in `Accounting/` and a CEO Briefing in `Briefings/`. Updates `Dashboard.md` with a direct link to the new briefing. No external API calls — all data comes from files already in the vault.

---

## Capabilities

| Capability | Command | Description |
|-----------|---------|-------------|
| Full audit | `run weekly audit` | Run all audit domains + generate briefing |
| CEO Briefing only | `generate CEO briefing` | Generate briefing from existing data |
| Task audit | `audit tasks` | Analyse `Done/` and `Needs_Action/` this week |
| Social audit | `audit social` | Read `Social_Summaries/` for the week |
| Accounting audit | `audit accounting` | Read `Accounting/` for the week |
| Approval audit | `audit approvals` | Check `Approved/`, `Rejected/`, `Pending_Approval/` |
| Error audit | `audit errors` | Scan `Logs/` for ERROR and ESCALATE entries |

---

## Inputs

- `week` (string, optional): ISO week `YYYY-WW` (defaults to current week)
- `domain` (string, optional): `personal | business | all` (default: `all`)
- `dry_run` (boolean, optional): Print report without writing files
- `force` (boolean, optional): Overwrite existing briefing for this week (adds `-r2` suffix)

| File | Location | Notes |

| `YYYY-WW — Accounting-Audit.md` | `Accounting/` | Financial summary for the week |

## Data Sources

| Audit Domain | Reads From | Extracts |
|-------------|-----------|---------|
| Tasks | `Done/YYYY-MM/` | Files created this week, tags, titles |
| Tasks | `Needs_Action/` | Open tasks, `due:` dates, `status:` |
| Social | `Social_Summaries/YYYY-WW — Social-Summary.md` | Posts count, platforms |
| Social | `Logs/YYYY-MM-DD.md` (SOCIAL_POST entries) | Fallback if no summary file |
| Accounting | `Accounting/*.md` for week period | Income, expenses, outstanding |
| Approvals | `Approved/`, `Rejected/`, `Pending_Approval/` | Counts, wait times |
| Errors | `Logs/YYYY-MM-DD.md` (ERROR, ESCALATE entries) | Error frequency, recurring failures |
| Briefings | `Briefings/` | Previous briefings for trend comparison |

---

## Steps

1. **Determine week range.** Calculate `YYYY-MM-DD to YYYY-MM-DD` from the ISO week.
2. **Task audit.** Count `Done/` files for the week. List titles. Count open `Needs_Action/` items by status and priority.
3. **Social audit.** Read `Social_Summaries/YYYY-WW — Social-Summary.md` or scan `Logs/` for `SOCIAL_POST` entries. Count posts per platform.
4. **Accounting audit.** Read `Accounting/*.md` within the date range. Sum income, expenses. List outstanding items. If `Accounting/` is empty → section says "No financial records this week."
5. **Approval audit.** Count approved/rejected/pending plans. Flag any plan in `Pending_Approval/` for >48 hours.
6. **Error audit.** Scan this week's `Logs/*.md` files for `ERROR`, `ESCALATE`, `LOOP_ABORT` entries. Count, group by skill.
7. **Write Accounting Audit.** Create `Accounting/YYYY-WW — Accounting-Audit.md` with frontmatter + financial summary table.
8. **Write CEO Briefing.** Create `Briefings/YYYY-WW — CEO-Briefing.md` with all 7 mandatory sections (see below). Set `status: pending-review`.
9. **Update Dashboard.md.** Prepend `[[Briefings/YYYY-WW — CEO-Briefing]]` under `## Latest Briefing` section.
10. **Log.** `AUDIT_RUN` then `BRIEFING_GEN`.

## CEO Briefing — 7 Required Sections

Every briefing generated by this skill MUST include these sections in this order:

```markdown
## 1. Executive Summary
3–5 bullet points. What happened this week at a glance.

## 2. Tasks Completed
- Total: N completed, N open, N blocked
- List of completed task titles (from Done/)
- Completion rate vs previous week if available

## 3. Social Media Activity
- Facebook: N posts | Instagram: N posts | Twitter: N posts | LinkedIn: N posts
- Any failures or escalations
- Link to Social_Summaries/YYYY-WW file

## 4. Accounting / Financial Summary
- Income this week: [amount or "No records"]
- Expenses this week: [amount or "No records"]
- Outstanding items: [list or "None"]
- Source: Accounting/ files

## 5. Pending Approvals & Blockers
- Plans waiting in Pending_Approval/ with age
- Any tasks with status: blocked
- Flag if any plan has waited >48 hours

## 6. Upcoming Deadlines
- Tasks with due: dates within next 7 days (from Needs_Action/)
- Sorted by due date ascending

## 7. Agent Recommendations
- 1–3 concrete actionable suggestions based on audit findings
- e.g. "3 recurring Twitter errors — consider re-authenticating the session"
```

## Acceptance Criteria

- [ ] Accounting audit written to `Accounting/` with correct frontmatter
- [ ] CEO Briefing written to `Briefings/` with `status: pending-review`
- [ ] All 7 sections present — empty sections say "No data this week" (not skipped)
- [ ] `Dashboard.md` updated with link to new briefing
- [ ] `force: true` creates `-r2` suffix, never overwrites existing briefing
- [ ] Audit runs without external API calls
- [ ] `AUDIT_RUN` and `BRIEFING_GEN` logged

## Constraints

- MUST NOT call external APIs to populate briefing
- MUST NOT overwrite existing briefing — use `-r2` suffix
- MUST NOT set `status: approved` on briefing — always `pending-review`
- MUST NOT include private message content (PII) in business domain sections
- MUST NOT skip sections — write "No data this week" if source is empty
